# aws-cloud-security-monitoring-system
Cloud security monitoring pipeline using AWS S3, Glue, Athena and Power BI to analyze cloud activity logs and detect suspicious behavior.

## Overview

This project demonstrates a cloud-based security monitoring pipeline built using AWS services to analyze cloud activity logs and detect suspicious behavior.

The project simulates how organizations monitor cloud environments to identify threats such as failed login attempts, unusual activity spikes, and abnormal access patterns.

## Architecture

Cloud Logs → Amazon S3 → AWS Glue Data Catalog → Amazon Athena → Power BI Security Dashboard

## Technologies Used

- Amazon Web Services (AWS)
- Amazon S3
- AWS Glue
- Amazon Athena
- SQL
- Microsoft Power BI

## Dataset

The dataset simulates cloud access logs similar to logs generated by AWS monitoring services.

Log attributes include:

- timestamp
- user_id
- ip_address
- country
- service_used
- action
- status
- bytes_transferred

Total records analyzed: 22,000+

## Security Analysis Performed

The following security monitoring queries were implemented:

- Failed login attempts per user
- Suspicious users with unusually high activity
- Most active IP addresses
- Activity distribution by country
- Unusual activity spikes over time
- Failed vs successful requests

## Dashboard Insights

The Power BI dashboard provides visual insights into:

- Failed vs successful access attempts
- Suspicious user activity
- Top IP addresses interacting with the system
- Geographic access patterns
- Activity trends over time
